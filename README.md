# リードα 復習セレクター

リードαの復習問題を、編・章・知・考・正答履歴から選ぶ静的Webアプリです。

## 現在の方針

- ユーザーが入力するのは4桁IDと誕生日だけです。
- ブラウザでのCSV読み込み・書き出しは使いません。
- 手動同期ボタンはありません。
- 記録など必要な操作が行われたときだけ、自動でクラウドへ保存します。
- 個人データはブラウザ上で暗号化してからGitHubへ保存します。
- 結果記録は履歴として複数積み上げます。
- 問題一覧のコメントは直近の記録コメントで上書きします。

## 管理者用クラウド設定

`app.js` の先頭付近にある `ADMIN_GITHUB` を設定します。

```js
const ADMIN_GITHUB = {
  owner: "あなたのGitHubユーザー名",
  repo: "保存先リポジトリ名",
  branch: "main",
  token: "GitHub fine-grained token",
};
```

保存先は `data/users/4桁ID.json` です。

注意: 静的サイトにGitHubトークンを直接入れると、利用者から見えます。自分だけで使う場合は簡単ですが、他人にも使わせる場合はCloudflare Workersなどの中継APIにトークンを置く方式にしてください。

## 管理用CSV

アプリ画面ではCSVを扱いません。初期データを整理するための管理用フォーマットだけ残しています。

- `lead-alpha-toc-template.csv`: 目次。列は `編,分野,章,名前,ページ`
- `lead-alpha-template.csv`: 問題。列は `編,章,種類,名前,知の指定,考の指定,関連問題,回数,正解数,最終実施日,コメント`
- `csv-format.md`: 列の説明
