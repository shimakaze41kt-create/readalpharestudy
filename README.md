# リードα 復習セレクター

リードαの復習問題を、問題の種類・関係性・回数・正答率から選ぶ静的Webアプリです。

## 現在の保存方式

- 4桁IDと誕生日でログインします。
- 学習データはブラウザ内に保存されます。
- 保存前に Web Crypto API で暗号化します。
- 管理者がコード側にGitHub保存先を設定すると、暗号化済みJSONだけを指定リポジトリに保存します。
- 別端末でも、4桁ID・誕生日だけでクラウドから復元できます。

## 注意

この方式は完全無料で運用しやすい一方、GitHubトークンをフロントエンドのコードに直接入れて公開すると、利用者から見えてしまいます。実運用ではCloudflare Workersなどの小さな中継APIにトークンを置く方式が安全です。

4桁IDと誕生日は覚えやすい反面、強い認証ではありません。公開範囲を広げる場合は、追加PINや通常パスワード方式への変更を推奨します。

## GitHub Pagesで公開する流れ

1. GitHubで新しいリポジトリを作る
2. このフォルダのファイルをリポジトリに追加する
3. GitHubの Settings > Pages で公開元を `main` ブランチにする
4. 表示されたURLを開く

## GitHub同期の準備

1. 学習データ保存用のリポジトリを作る
2. GitHubの Developer settings で fine-grained personal access token を作る
3. 対象リポジトリだけに `Contents: Read and write` 権限を付ける
4. `app.js` 冒頭の `ADMIN_GITHUB` に保存先を設定する
5. アプリでは利用者が4桁ID、誕生日、名前だけを入力する
6. 自動で暗号化データをアップロードする

保存先は自動指定です。

- 学習データ: `data/users/4桁ID.json`
- CSVバックアップ: `data/users/4桁ID.csv.enc.json`

どちらも暗号化済みJSONです。4桁IDと誕生日がないと復号できません。

GitHub設定を保存した後は、結果記録、CSV読み込み、サンプル戻しのたびに自動でGitHubへ保存します。通信に失敗した場合でも、端末内の暗号化保存は残ります。

CSVは学習データを保存するたびに端末内で自動生成され、GitHub同期時にはクラウドにも暗号化バックアップされます。「CSVを書き出す」は最新の自動生成CSVを取り出すためのボタンです。ブラウザの安全制約により、端末上の任意ファイルを完全自動で読み込むことはできないため、CSVを取り込むときだけファイル選択が必要です。

## 完全クラウド復元

別端末やブラウザ初期化後は、ログイン画面で次を入力します。

- 4桁ID
- 誕生日

そのまま「ログイン」を押すと、`data/users/4桁ID.json` から暗号化データを取得して復元します。

## 管理者用設定

`app.js` の先頭付近にある次の値を設定します。

```js
const ADMIN_GITHUB = {
  owner: "あなたのGitHubユーザー名",
  repo: "保存先リポジトリ名",
  branch: "main",
  token: "GitHub fine-grained token",
};
```

公開ページに直接トークンを入れると、ブラウザの開発者ツールから見えます。自分だけが使うローカル用途なら簡単ですが、他の利用者へ公開するなら中継API化してください。

## CSV

- `lead-alpha-template.csv`: アプリに読み込む学習データ
- `lead-alpha-relations-format.csv`: 問題同士の関係を整理するための一覧
- `csv-format.md`: CSV列の説明
